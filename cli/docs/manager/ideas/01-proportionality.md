# 01 — Proportionality: build lean, defer the rest

> This is the single most important constraint in the corpus. It overrides ambition. A red-team review
> of the full six-dimension design returned a blunt verdict, and we accept it.

## The verdict

> For a **solo developer's personal harness at `0.1.0-design` (~93 files, a handful of tailored
> projects)**, the full six-dimension manager is *a governance platform cosplaying as a personal tool*.
> The manager risks **doubling the codebase to manage the original half** — and must then register and
> eval *itself*, recursively. The proportionate version is **a few validator extensions, a registry for
> identity, a JSON flag, and a dashboard** — with **every gate advisory, never blocking**.

We design the *full* conceptual model (so it is coherent and growable), but we **build** only the lean
core now, and we keep gates advisory until real data justifies hardening them.

## Three tiers

### Tier 1 — Build now (the lean core)
The identity + visibility spine. Cheap, low-ceremony, high-leverage, and a prerequisite for everything.

- **`--json` backbone** at the parent runner (`ADR-0004`) — touches no child validator; makes the whole
  harness machine-readable. *Highest leverage single change.*
- **Registry** (`SPEC-01`) — catalog + `contentHash` identity for all artifacts; fixes the `VERSION`
  drift by reading it from one place.
- **Dependency graph, prose-ref subset** (`SPEC-03`) — catches the real `react-reviewer` dangling
  reference that `validate-xref` misses today. This is an *actual recurring bug class* (the
  "13 broken skill links" defect class), not a hypothetical.
- **`forge status` skeleton** (`SPEC-08`) — composes the above; proves the dashboard contract.

### Tier 2 — Grow into (when a specific pain appears)
Real engineering, real payoff, but only once the trigger condition is met.

- **Context-budget accounting** (`SPEC-06`, static half) — trigger: context feels heavy / you want to
  trim always-on cost. Cheap and data-free; can come early.
- **Eval-of-harness as pass/fail meta-tests** (`SPEC-07`) — trigger: you edit a reviewer and want proof
  it still catches planted defects. Keep the planted-defect + clean-code cases; **drop the A–F GPA**.
- **Telemetry** (`SPEC-05`) — trigger: you genuinely can't tell which artifacts you use. For `n=1`, you
  often already know; defer until you don't.

### Tier 3 — Defer until many-projects or a collaborator (probably-never at current scale)
- **Fleet index + bulk 3-way sync** (`SPEC-04`) — `find ~ -name .forge.json` already enumerates; git
  already distributes. Build the index only when you have enough tailored projects that manual is painful.
- **Value-density optimization** (`SPEC-06`, dynamic half) — `effScore / costTok` is a ratio of an
  LLM-judged score over an *estimated* token count; at `n=1` it is "noise wearing a lab coat" until both
  inputs are real and plentiful.

## Two corrections the red-team forced (baked into the ADRs)

1. **Live-symlink seam** (`ADR-0008`) — `forge install` symlinks the library *live*, so an artifact's
   content can change with **no install event** to hang a revision bump on. Therefore the registry is
   **regenerated by scan-on-demand**, and "hash changed but revision didn't" is a **WARN**, never a
   commit block.
2. **Gates are advisory** (`ADR-0007`) — a blocking version-bump gate × a blocking eval-regression gate
   is a *deadlock surface* for a team of one, and the solo dev's escape hatch (`git --no-verify`) is the
   very thing forge's own `block-no-verify` hook fights. So both ship as `WARN` findings and only become
   blocking once eval data exists to justify the block — your explicit decision, later.

## The discipline, stated as a rule

> **The manager must never cost more to maintain than the harness it manages.** Every dimension ships
> only when its trigger condition is met; every gate is advisory until proven worth blocking; everything
> degrades gracefully (cost-only, static-only, grade-`U`) when its upstream signal is absent.

This rule is encoded normatively as `BR-INT` and gated by the phasing in `ROADMAP.md`.
