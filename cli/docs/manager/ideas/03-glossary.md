# 03 — Glossary

Terms used across the corpus. Where a term already has a meaning in forge, this preserves it.

- **Artifact** — a managed unit of the harness: an agent, skill, command, rule, hook, bundle, validator,
  meta-test, or engine script. Identified by `uid = "<kind>:<id>"` (e.g. `agent:code-reviewer`).
- **Global harness / library** — the forge plugin payload (source of truth): `agents/ skills/ commands/
  rules/ hooks/ bundles/`, `manifests/`, `bin/forge.mjs`, `lint/`, `tests/meta/`.
- **Tailored harness** — a per-project `.claude/` generated by `forge init`; references global
  agents/skills, copies rules + customized files, and carries a **marker**.
- **Marker** — `<project>/.claude/.forge.json`: `{forgeVersion, profile, modules[], generatedAt,
  files[]{path, checksum, userEditable}}`. The per-project source of truth; the manager *extends* it with
  one `provenance.sourceRev` field (`ADR-0009`).
- **Registry** — `forge/.forge/registry.json`: the global catalog of all artifacts with identity,
  version, status, dependency edges, and an eval-linkage slot. Git-tracked, regenerated by scan.
- **`contentHash`** — `sha256:…` over an artifact's canonical bytes. The **sole identity primitive**
  (`ADR-0005`); everything (drift, provenance, eval staleness) derives from it.
- **`revision`** — a monotonic integer, `+1` per accepted `contentHash` change. Cheap ordering cursor.
- **`version`** — per-artifact semver (`MAJOR.MINOR.PATCH`): human-meaningful intent of a change.
- **`VERSION` roll-up** — forge's single release version, computed as a deterministic fold over artifact
  revisions/hashes rather than hand-maintained (`ADR-0006`).
- **`sourceRev`** — a marker field: `sha256` over the sorted `{uid: contentHash}` of the components a
  project's modules resolve to, at tailor time. One cheap hash answers "is this project in sync?".
- **Fleet** — the set of tailored projects; indexed (opt-in, cached) in `~/.claude/forge/fleet.json`.
- **Drift** — divergence between a tailored project and the current global harness: *version-behind*,
  *components-behind*, *user-edited files*, *broken refs*.
- **Telemetry event** — one JSONL line capturing a hook fire/decision, an invocation start, or a
  validator/eval run; redacted on write (`ADR-0011`).
- **Context-budget** — the estimated token cost an artifact imposes: *always-on* (rules with no `paths:`,
  agent/skill descriptions, hook injections) vs *conditional* (path-scoped rules, bodies, bundles).
- **Criticality** — an artifact tag `safety | compliance | normal`. `safety`/`compliance` artifacts are
  **never** pruned regardless of usage (`ADR-0013`). A `secret-scan` that fires 0× is *working*, not waste.
- **Value-density** — `effectiveness / context-cost`; a pruning heuristic. Deferred (Tier 3) because both
  inputs are weak at `n=1`.
- **Tier S / Tier B** — two tiers of *forge-validates-forge*: Tier S = static prose meta-tests (exists
  today); Tier B = behavioral evals on planted-defect golden sets (`SPEC-07`). Tier S gates Tier B.
- **Catch-rate / false-positive rate** — for a reviewer: fraction of planted defects found / fraction of
  clean code wrongly flagged. The anti-noise value, made measurable.
- **Grade `U` (unevaluated)** — an honest non-score: an artifact with no golden set. Never coerced to 0
  or 1; counts against *coverage*, not health (`ADR-0012`).
- **Advisory gate** — a check that emits a `WARN` finding but does not block a commit/CI (`ADR-0007`).
- **Findings shape** — the unified `{level, path, line, message, source}` every check emits; the basis of
  the `--json` envelope (`ADR-0004`).
- **Envelope** — the one machine-output shape: `{forge, command, ok, ts, data, findings[], summary}`.
- **Manager module** — `forge/manager/<noun>.mjs`, exporting `run()` + `summarize()`, fail-open, dry-run
  by default, with a paired auto-discovered `lint/validate-<noun>.mjs` (`C4`, `BR-INT`).
