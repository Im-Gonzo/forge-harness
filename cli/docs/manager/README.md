The **Harness Manager** is the management layer over forge: the *temporal + identity + observability*
dimension forge lacks today. Forge currently is a strong **static integrity verifier** (composition
graph, cross-references, governance prose, CI-gated) but has no sense of *what changed, what an artifact
costs, whether it still works, where it is deployed, or whether it earns its keep*. The manager supplies
exactly that missing layer.

Start with `ideas/00-concept.md` for the thesis and `ideas/01-proportionality.md` for the scope verdict
(the single most important design constraint: **build lean, keep gates advisory, grow into the rest**).

## How this corpus is organized

| Dir | What | Convention |
|---|---|---|
| `ideas/` | Concept, proportionality verdict, scope/non-goals, glossary | prose |
| `adr/` | Architecture Decision Records | `ADR-NNNN-slug.md`, ID `ADR-NNNN` |
| `business-rules/` | Normative rules, one file per dimension | `BR-<PREFIX>-NNN` |
| `spec/` | Detailed design, one file per dimension | `SPEC-NN` |
| `evals/` | Acceptance specs (RED-first), one file per dimension | `EVAL-<PREFIX>-NNN` |
| `ROADMAP.md` | Phased build (v0.2 ‚Üí v0.6) with definition-of-done | ‚Äî |

**Traceability is the point.** Every BR cites the ADR(s) that decided it and the SPEC that details it;
every BR has an acceptance criterion that maps to one or more EVAL cases; development for a phase is
"done" when that phase's `MUST` EVAL cases are GREEN. `BR <-> ADR <-> SPEC <-> EVAL` is a closed loop.

## House style (every doc conforms)

**ADR** ‚Äî `# ADR-NNNN: Title` then `Status` (Accepted (design-stage) | Proposed | Superseded), `Date`
(2026-06-05), `Phase` (v0.x); sections `## Context`, `## Decision`, `## Consequences` (positive /
negative / neutral), `## Alternatives considered`, `## Related` (ADR/BR/SPEC IDs).

**BR file** ‚Äî `# Business Rules ‚Äî <Dimension> (BR-<PREFIX>)`, then one block per rule:
`### BR-<PREFIX>-NNN ‚Äî short title` with `**Rule:**` (normative MUST/SHOULD/MAY), `**Rationale:**`,
`**Acceptance:**` (testable; names the `EVAL-‚Ä¶` case), `**Priority:**` (MUST|SHOULD|MAY),
`**Refs:**` (ADR/SPEC IDs).

**SPEC** ‚Äî `# SPEC-NN ‚Äî Title`; header line `Status ¬∑ Phase ¬∑ Implements: BR-‚Ä¶ ¬∑ Decided-by: ADR-‚Ä¶`;
sections `## Summary`, `## Design`, `## Data structures`, `## CLI / interface`, `## Edge cases &
failure modes`, `## Open questions`, `## Traceability` (BR/ADR/EVAL).

**EVAL file** ‚Äî `# EVAL-<PREFIX> ‚Äî <Dimension> acceptance specs`; one block per case:
`### EVAL-<PREFIX>-NNN ‚Äî title` with `**Verifies:**` (BR-IDs), `**Kind:**` (capability|regression),
`**Grader:**` (code|model|human), `**Target:**` (e.g. `pass^k=1.00`), `**Given/When/Then:**`,
`**Fixture:**`, `**Phase:**` (v0.x), `**Status:**` (RED ‚Äî none are implemented yet).

## Foundational invariants (inherited from forge; non-negotiable)

1. **Zero runtime dependencies** ‚Äî `node:` builtins only (`>=18`; assume `>=22` features only if guarded).
2. **Additive, never destructive** ‚Äî writes skip/preserve existing files; user edits are sacred.
3. **Detect-and-offer, never auto-mutate** ‚Äî global state changes require explicit confirmation.
4. **Fail-open** ‚Äî a manager/telemetry failure never blocks a tool call or a session.
5. **Forge-validates-forge** ‚Äî the manager ships its own auto-discovered validators.
6. **Local-only / privacy** ‚Äî no exfiltration; machine-local data physically cannot be committed.

## The cross-dimension contract (one-line invariants)

- **C1 Identity** ‚Äî `contentHash` (sha256, via one shared hasher) is the only identity primitive; fleet
  `sourceRev`, efficiency keys, and eval `pinnedHash` all derive from it. (ADR-0005)
- **C2 Findings** ‚Äî every check emits `{level, path, line, message, source}`; child validators print
  `LEVEL path:line message`, the **parent** runner parses them into `findings[]`. (ADR-0004)
- **C3 Envelope** ‚Äî machine output is one shape: `{forge, command, ok, ts, data, findings[], summary}`.
- **C4 Module contract** ‚Äî each `forge/manager/<noun>.mjs` exports `run(subcmd, args, ctx)` and
  `summarize(state)`; reads/writes state only via `manager/lib/store.mjs`; is fail-open; dry-run by
  default; ships a paired `lint/validate-<noun>.mjs`. (ADR-0014, BR-INT)
- **C5 Gates** ‚Äî version-bump and eval-regression checks are **advisory (WARN)** until eval data exists
  to justify blocking. (ADR-0007)
- **C6 Storage split** ‚Äî git-tracked truth in `forge/.forge/`; machine-local cache in `~/.claude/forge/`;
  no machine-local file is ever authoritative. (ADR-0003)
