# Forge schemas

Draft-07 JSON Schemas that pin the shapes Forge depends on, checked by Forge's zero-dependency `validate-manifests`/`bundle-lint` validators (not AJV). `marker.schema.json` validates the per-project idempotency marker `<project>/.claude/.forge.json` (BOOTSTRAP §Mark); `profile-project.schema.json` validates the deterministic profiler output `<project>/.claude/profile-project.json` (BOOTSTRAP §Profile); `profiles.schema.json` and `modules.schema.json` validate the composition manifests in `manifests/`; `hooks.schema.json` is a minimal schema for a Claude Code plugin `hooks.json`; and `bundle.schema.json` validates a context bundle's frontmatter (the 16 required keys and the B-1..B-4 invariants from `bundle-lint`). Field names here are load-bearing — the CLI (`bin/forge.mjs`), the `bootstrap-harness` skill, and the validators all key off them.
